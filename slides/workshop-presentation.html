Part 1: The Interactions API

One API, Two Modes

Unified Interface
  • Models: call any Gemini model with model=
  • Agents: invoke built-in agents with agent=
Same Surface for Everything
  • input: text, typed content blocks, or turns
  • tools: custom functions + built-in tools
  • stream: real-time SSE events
  • background: async execution
Today: Deep Research agent. Tomorrow: your own agents.
Request Shape
{ "model": "gemini-3-flash", // or "agent": "deep-research" "input": "Your prompt", "tools": [...], "stream": true, "background": true, "previous_interaction_id": "..." }
POST /v1beta/interactions
Part 1: The Interactions API

Data Model: Content Blocks

Input Content Blocks
  • text: plain text prompt or follow-up
  • image / audio / video / document: multimodal data or URI
  • function_result: result with call_id + result
  • thought: previous thought with signature
Output Content Blocks
  • text: model response with optional annotations
  • function_call: tool invocation: name, arguments, id
  • image / audio: generated media
  • google_search_call/result, url_context_call/result, code_execution_call/result
  • thought: reasoning trace with signature
Flat typed blocks: no nesting
// Response: flat array of typed blocks { "status": "requires_action", "outputs": [ { "type": "thought", "signature": "abc123..." }, { "type": "function_call", "id": "gth23981", "name": "get_weather", "arguments": {"location": "Boston"} } ] }
Every block is self-describing, check output.type to route processing.

Core Primitives

Server-Side State
i2 = client.interactions.create( model="gemini-3-flash-preview", input="Follow up question", previous_interaction_id=i1.id )
Chain turns with previous_interaction_id. Server owns conversation history.
Background Execution
r = client.interactions.create( agent="deep-research...", input="Research topic", background=True )
Fire-and-forget for long tasks. Poll with interactions.get(id).
Typed Output Blocks
for o in interaction.outputs: match o.type: case "text": print(o.text) case "function_call": ...
Flat array: text, function_call, image, thought.
Streaming via SSE
for chunk in client.interactions.create( model="gemini-3-flash...", input="Hello", stream=True ): print(chunk)
Rich events: content.delta, interaction.complete.

Part 2: Building Agents

What Actually is an Agent?

  • The Model (Brain): Reasoning engine that plans and decides when to use tools
  • Tools (Hands & Eyes): Tools to interact with the environment
  • Context (Workspace): Information at each step.
  • The Loop (Life): while loop: Observe → Think → Act → Repeat
If you can write a while loop in Python, you can build an agent.
Part 2: Building Agents

Using the API: Basic Chat

Stateful Multi-Turn
  • Create an interaction with a prompt
  • Chain turns with previous_interaction_id
  • Zero history: server manages all state
No contents=[] array to build. Just send the next message.
from google import genai client = genai.Client() # Turn 1 i1 = client.interactions.create( model="gemini-3-flash-preview", input="What is the capital of France?" ) print(i1.outputs[-1].text) # "Paris" # Turn 2: server remembers turn 1 i2 = client.interactions.create( model="gemini-3-flash-preview", input="And what is the population?", previous_interaction_id=i1.id ) print(i2.outputs[-1].text)
Part 2: Building Agents

Swap Agents & Models Mid-Conversation

One Conversation, Two Backends
  • Turn 1: call agent= for Deep Research
  • Turn 2: switch to model= for Nano Banana
  • Same chain: linked via previous_interaction_id
Agents and models are interchangeable: the server-side state carries across both.
from google import genai client = genai.Client() # Turn 1: Deep Research agent i1 = client.interactions.create( agent="deep-research-pro-preview-12-2025", input="Research AI agents in 2026", background=True ) # ... poll until i1.status == "completed" # Turn 2: Nano Banana model on same chain i2 = client.interactions.create( model="gemini-3.1-flash-image-preview", input="Generate a infographic for the report", previous_interaction_id=i1.id ) print(i2.outputs[-1].text)

Using the API: Tool Use

Function Calling
  • Define tools as typed declarations
  • Check outputs for function_call blocks
  • Return results as function_result
  • Status is requires_action when tools are needed
tools = [{ "type": "function", "name": "get_weather", "description": "Get current weather", "parameters": { "type": "object", "properties": { "location": {"type": "string"} } } }] interaction = client.interactions.create( model="gemini-3-flash-preview", input="What's the weather in Paris?", tools=tools ) # interaction.status == "requires_action" # interaction.outputs[0].type == "function_call" # interaction.outputs[0].name == "get_weather" # interaction.outputs[0].arguments == {"location": "Paris"}
Part 2: Building Agents

The Agent Loop

How it works
  • Send request: prompt + tool definitions
  • Check outputs: look for function_call blocks
  • Execute locally: run your function code
  • Send results back: as function_result
  • Repeat: until model returns text
No history managementServer tracks via interaction ID
Thought signaturesAutomatic via previous_interaction_id
# Initial call with tools interaction = client.interactions.create( model="gemini-3-flash-preview", input="Read example.txt and summarize it", tools=tools ) # Agent loop: keep going while model needs tools while interaction.status == "requires_action": results = [] for output in interaction.outputs: if output.type == "function_call": result = execute(output.name, output.args) results.append({ "type": "function_result", "name": output.name, "call_id": output.id, "result": result }) interaction = client.interactions.create( model="gemini-3-flash-preview", previous_interaction_id=interaction.id, input=results, tools=tools )

Deep Research: Background Agent

Same API, Different Mode
  • Switch model= → agent=: invoke built-in autonomous agent
  • Set background=True: no client connection required
  • Poll for completion: interactions.get(id)
# Launch Deep Research research = client.interactions.create( agent="deep-research-pro-preview", input="Research AI agents in 2026", background=True ) # Poll for completion while True: result = client.interactions.get( research.id ) if result.status == "completed": print(result.outputs[-1].text) break time.sleep(10)
